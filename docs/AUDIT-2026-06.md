# Аудит seedream-bot (Hitek5/seedream-bot, public)

## 1. Архитектура

Стек: Node.js (tsx, без сборки), TypeScript strict, ESM. TG — grammY (^1.35, lock 1.41.1). fal.ai — @fal-ai/client 1.9.5, Anthropic SDK 0.82 (Claude Vision + кодоген CAD). Точка входа: src/index.ts → createBot() (src/bot/index.ts). Long polling, webhook нет.

Сервисы: seedream.ts (t2i + edit, fal-ai/bytedance/seedream/v4.5/*), vision.ts (Claude Vision, fallback Florence-2 на fal), claude.ts, threed.ts (birefnet + TripoSR/Hunyuan3D/Trellis), cad.ts (Claude генерит CadQuery-Python → execFile python3), balance.ts.

Хранилища: data/balances.json (sync-запись) + in-memory Map (promptStore, pendingFaceSwap — callback.ts:14,30). SQLite из плана не реализован.

Деплой: в репо нет (README декларирует PM2+systemd); коммиты «auto-backup» с меняющимся balances.json — прод работал из рабочей копии.

## 2. Карта процессов для BPMN

Дорожки: Пользователь / Бот (grammY) / fal.ai / Anthropic / Telegram API / Python (CadQuery).

P1. Текст → картинка (index.ts:193 handleGenerate): Старт(message:text) → XOR «промпт пуст?» [да → подсказка → Конец] → XOR «баланс ≥ $0.04?» [нет → «Недостаточно средств» → Конец] → chatAction + цикл-таймер 4с (имитация прогресса) → fal.subscribe(t2i) (submit → внутренний polling → результат; без явного timeout, без onQueueUpdate, без ретраев) → XOR «успех?»: да → deductBalance (ПОСЛЕ генерации) → promptStore → replyWithPhoto + кнопки (Ещё вариант / Изменить / С моим фото / Сохранить / Сначала); нет → getErrorMessage (index.ts:279): подшлюз timeout / NSFW(«safety|nsfw|content_filter») / rate limit(«rate|429») / прочее → ответ → Конец. Петля «Ещё вариант» = повтор P1 с промптом из Map/caption (seed пишется, но не переиспользуется).

P2. Фото → промпт (photo.ts): Старт(message:photo) → XOR «pendingFaceSwap?» [да → P3] → getFile → URL api.telegram.org/file/bot<TOKEN>/… → analyzeImage: XOR «ANTHROPIC_API_KEY?» → Claude Vision (claude-sonnet-4-20250514); ошибка → fallback Florence-2 (fal) → «промпт + 7 кнопок» → Конец. Анализ бесплатный, без лимитов.

P3. Face swap (callback.ts): Старт(callback face_swap) → XOR «промпт найден?» (Map → fallback caption) → запрос фото → Старт(фото) → Florence-2 extractHairColor (ошибки глотаются) → клавиатура параметров → цикл: callback fs_* → XOR «рост И фигура выбраны?» → XOR баланс ≥ $0.08 → uploadToFal (fetch TG-URL → fal.storage) → fal.subscribe(edit) → списание → фото+кнопки → Конец / ошибка → сброс → Конец.

P5. 3D: callback threed_menu → XOR качество (fast/standard/high) → баланс → removeBackground (birefnet) → generateThreeD → списание → GLB → Конец.

P6. CAD: /cad или callback → баланс → XOR «CadQuery установлен?» [нет → подсказка → Конец] → (фото → Claude Vision описание) → Claude генерит Python → tmp → execFile python3, таймаут 60с → XOR «файлы созданы?» → STEP+STL → списание → cleanup → Конец.

Рестарт посреди генерации: update передоставляется (возможна повторная генерация = двойной расход fal), in-flight теряется, списания нет (после успеха); Map-стейты теряются (частично спасает caption-fallback).

## 3. Аудит

Безопасность: история git ЧИСТАЯ (токенов нет, .env не коммитился). ❌ Главное: BOT_TOKEN уходит на fal.ai в составе file-URL (photo.ts:16,46 → vision.ts:135,178; callback.ts:165 → threed.ts:28) — третья сторона получает токен = полный захват бота. ❌ RCE by design: cad.ts:107-144 исполняет LLM-Python без песочницы (промпт контролирует пользователь). Whitelist де-факто: новый юзер = баланс 0 (генерация недоступна), НО vision-анализ фото бесплатный и нелимитированный (жжёт Anthropic/fal). ADMIN_IDS и MAX_DAILY_IMAGES читаются из config (config.ts:15,19) и МЕРТВЫ — лимитов и админ-команд пополнения нет (баланс правится руками в JSON). PII: data/balances.json с TG-ID владельца в публичном репо (история). TOCTOU-гонка баланса; deductBalance=false игнорируется.

Зависимости: свежие (grammy 1.41.1, @fal-ai/client 1.9.5, SDK 0.82, TS 5.9); модель claude-sonnet-4-20250514 захардкожена (claude.ts:39,76).

Надёжность: bot.start() без runner — апдейты ПОСЛЕДОВАТЕЛЬНО (одна CAD-генерация 60с блокирует всех); нет unhandledRejection-обработчика; fal.subscribe без таймаута; promptStore/pendingFaceSwap растут бесконечно; writeFileSync неатомарен; битый JSON молча обнуляет балансы (balance.ts:35-39).

Тесты/линт: нет; tsc strict (но ctx:any в index.ts:193).

Деплой: не формализован; логи console.log.
